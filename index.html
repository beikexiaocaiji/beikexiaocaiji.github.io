Java多线程

Posted on 2020-07-16

Java多线程
1.进程与线程
程序是静态的概念
（1）进程就是正在执行的程序，比如说使用QQ
（2）线程就是进程更细的划分，比如说QQ里面的音乐，视频，聊天等等。
一个进程就占有一个方法区和堆，各个线程共享，也就是一个进程有多个线程并行。
线程各自有自己的虚拟机栈和程序计数器，共享资源因此相互通信很方便，但是访问资源是并发，有安全隐患。
引入：并发和并行
并发，多个人干一件事；
并行，多个人干多件事；
（3）多线程的意义
进程与进程之间切换开销大，但是线程之间开销小，可以减少CPU切换开销。
同时CPU多核处理多线程，效率也会提高。
一个Java程序，至少三个线程：main方法，异常处理，垃圾回收线程。

2.线程的启用
（1）继承Thread类
步骤：继承Thread类，重写run方法，创建对象调用start方法
//匿名内部类就是直接用构造器，{}内添加自己写的东西，没有对象名字
new Thread(){
@Override
public void run() {
System.out.println(“hhh”);
}
}.start();
注意：对象直接调用run方法不会产生多线程，多次调用start方法会报错
（2）实现Runnable接口
步骤：实现Runnable接口，实现run方法，创建线程对象，再将线程对象作为参数传入Thread类构造器，调用start方法。
class MyThread implements Runnable{
@Override
public void run() {
System.out.println(“jjj”);
}
}
//这是静态代理模式，MyThread和Thread类都实现Runnable接口
//创建目标对象
MyThread myThread = new MyThread();
//创建代理对象
Thread thread = new Thread(myThread);
thread.start();
（3）创建方式的比较
继承Thread类有局限性，无法继承其他类，
实现Runnable接口更灵活，同时可以创建多个代理对象，天然共享Runnable对象数据不用加static
（4）线程的优先级
注意线程的优先级表示的是概率，优先级高抢占CPU的概率高，同时不一定优先级高要完成以后才做优先级低的
（5）线程的状态
一定要画图分析清楚，什么方法，什么情况会转换状态
初始态（NEW）:创建对象，还没调用start方法
就绪台（READY）：调用start方法，只要有CPU执行权就可以执行
运行态（RUNNING）：有CPU执行权，正在执行
等待态（WAITING）：
超时等待态（TIME_WAITING）：
阻塞态（BLOCKED）：
终止态（TERMINATED）：线程运行完毕
（6）线程的分类
分为守护线程和用户线程。守护线程是为用户线程服务，不会单独存在。
例如main方法的主线程就是用户线程，GC垃圾回收线程就是守护线程

3.线程安全问题：
概述：一个线程在操作某个资源的过程中还没结束，另一个线程进入。导致共享数据错误
解决:
(1)synchronized
synchronized可以同步很多，根据是作为锁的对象不同
主要是同步代码块和同步方法
一定要注意锁定的是对象而不是代码，多线程只有拿到同一个对象，同步才生效！！
同步代码块：
private int ticket = 100;
//锁对象不能为NULL
private Object object = new Object();

        @Override
        public void run() {
            while (true){
            //synchronized (MyThread.class)通过当前类.class充当对象比较方便！！
            //this对象继承类实现的线程，同步也不会生效
                synchronized (object){
                    if (ticket > 0){
                        System.out.println(Thread.currentThread().getName() + " " + ticket);
                        ticket--;
                    }
                }
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    同步方法：只需要在方法上加synchronized关键字
    其实其中也有锁：普通方法是调用run方法的当前对象，静态方法是当前类.class
                注意this对象继承类实现的线程，同步也不会生效

    引入单例模式懒汉式线程安全解决办法：
    （1）同步方法，效率较差
public static synchronized Singleton getSingleton(){
    if (singleton == null){
        singleton = new Singleton();
    }
    return singleton;
}

    （2）同步代码块，加上进入其之前的判断，效果较好
public static Singleton getSingleton(){
    if (singleton == null) {
        synchronized (Singleton.class) {
            if (singleton == null){
                singleton = new Singleton();
            }
        }
    }
    return singleton;
}    

（2）lock机制解决
JDK5以后新增，也可以解决同步问题
步骤：创建ReentrantLock对象，调用lock方法上锁，紧跟要同步try块，finally加上unlock方法解锁
private ReentrantLock lock = new ReentrantLock();
@Override
public void run() {
    while (true){
        lock.lock();
        try {
            if (ticket > 0){
                System.out.println(Thread.currentThread().getName() + " " + ticket);
                ticket--;
            }
        } finally {
            lock.unlock();
        }
    }
}
（3）lock与synchronized的区别
相同：两者都能实现互斥和重入（也就是加锁之后解锁之前，已得到该锁的单线程可以调用使用同一把锁的其他方法，
此时线程就会持有多把相同的锁）
区别：synchronized是JVM实现，lock是JDK实现
    lock有其他更高级的用法（如公平锁，释放锁之后让等待线程先到先得），但是基本功能还是用synchronized
    因为后者能自动释放锁

4.死锁问题
线程之间相互占有对方需要的资源，拿着对方需要的锁（好比我拿着你的筷子，你拿着我的筷子，大家吃不了饭，释放不了筷子）
造成等待浪费资源

复用类

Posted on 2020-07-07

复用类的方式有：组合，继承和代理（中庸之道）

对象中域的基本类型自动初始化为0，对象引用为自动初始化为NULL（对象内存设为0一举实现），因为可以减轻内存负担。初始化方式有四种：（1）定义时初始化（2）构造器初始化（3）实例初始化（用的少）（4）在使用前初始化（惰性初始化）

继承意味着导出类复用基类的接口（一般情况数据成员private，方法为public），导出类对象构造器会首先调用基类构造器产生内部基类对象（Java默认自动调用基类默认构造器，想调用基类带参数构造器需要用super）

代理可以通过代理对象控制原对象的接口，只暴露他的一个子集而不会像继承一样暴露全部。(idea 快捷键Alt+Ins)

注意导出类不会屏蔽基类的重载方法，因此要注意导出类是重载还是覆写，覆写要加@Override注解

组合表达has a关系，继承表达is a关系，优先使用组合（因为其更灵活），判断方法是到底需不需要向上转型。

final

final数据：（1）基本类型：典型的public static final表示编译时常量，大写字母命名（下划线分隔）（2）对象引用：表示引用恒定（无法再改为另一个对象，但是对象内容可以修改）
final数据必须在定义或构造器（可提供更大的灵活性）赋值， final参数意味着你无法在方法中改变引用的对象（对象引用）或修改参数（基本类型）

final方法：意味着不能覆写，private方法隐式表达final（因为不能获取怎么能覆写）

final类：表明不能继承。

类在初次使用才加载（访问任何static成员时加载）。运行main方法（其是static），加载当前类，再通过extends加载其基类，以此类推（这种加载方式很重要，因为导出类的static初始化可能依赖于基类正确初始化）

反射机制

Posted on 2020-07-04

一般情况是写代码 -> 编译 -> 运行。反射则是在运行时获得类的内部信息，操作类的内部方法和属性，形象地称为反射，在编译时不知道具体类使用，因而常用在通用性很高的框架。

运行时类信息存储在Class对象中，每个类的.class文件都会被JVM“类加载器”子系统加载到内存中。注意是程序创建第一个对类的静态成员的引用时动态加载（说明构造器也是静态方法）

反射机制使用Class类主动创建Class对象，而传统情况感受不到Class对象。Class类有多种方法创建Class对象，（1）getClass()已持有该类对象（2）forName()（3）.class类字面常量（这种方法创建Class对象时不会自动初始化，JVM“类加载器”子系统加载到内存过程包括加载、链接和初始化）

有了Class对象，newInstance()创建实例（类必须有默认构造器，避免调用失败权限应为public），因此本质上还是调用构造器。

建议使用泛型替代原生Class引用，newInstance()返回对象确切类型。必要时类型转换（不知道对象确切类型时），用instanceof()来判断。
