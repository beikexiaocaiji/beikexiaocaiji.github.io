关键字

Posted on 2020-07-21

关键字：
1.final：
final可用于属性，方法和类当中：
（1）基本数据类型:表示赋值了以后不能更改（有最终的意思），相当于只能赋值一次，常用于表示常量
引用类型：表示引用指向对象这个关系不可以再修改（只能指向那个对象），但是对象的内容可以改
（2）方法：不可以重写
（3）类：不可以继承，不想别人改掉你的接口，如String类

2.this&super
（1）this，这个，表示当前的对象，常用于getter、setter赋值操作；也用在复用自己的构造器
（2）super，父类的对象，常用于构造器，因为构造器需要先初始化父类的对象，
因此肯定要调用父类的构造器，父类默认构造器自动调用，有参数构造器就用super调用；

3.abstract
(1)方法：说明是抽象方法，只有定义没有实现，实现交给导出类，有抽象方法肯定是抽象类
（2）类：说明是抽象类，表示抽象的概念，不能实例化

4.访问权限：
（1）private：其他所有人都不可以访问
（2）package、无：表示只有同一个包里的类可以访问
（3）protected：在包访问的基础上加上导出类可以访问
（4）public：所有人都可以访问

5.static关键字
static关键字的含义：
随着类的加载而加载，不与实例对象有关，被对象所共享，可用来修饰属性，方法，内部类和字段
（1）静态属性：常表示常量，被对象所共享。存储在方法区，只有一份，可以通过类名或引用来修改它（常量不可以）
（2）静态方法：可以用来调用静态属性。常用在工具类，这样不需要创建工具类对象
static方法不能调用实例方法，以及与具体对象相关的（this、super等），
但是实例方法可以调用static方法。
引入单例模式：
饿汉式：
public class Singleton(){
//隐藏构造器
private Singleton();

    //只有一份且被共享
    private static Singleton single = new Singleton();
    //静态方法可以用来调用静态属性
    public static Singleton getSingleton(){
        return single;
    }
}

懒汉式：
public class Singleton(){
    //隐藏构造器
    private Singleton();

    //只有一份且被共享
    private static Singleton single = null;
    //静态方法可以用来调用静态属性
    public static Singleton getSingleton(){
        if(single != null){
            //判断与赋值分离之间，并发可能会有两个创建
            single = new Singleton();
        }
        return single;
    }
}    
比较：懒汉式对象延迟加载，但是非线程安全
（3）静态内部类
（4）静态代码块：随着类的加载而加载，通常用于初始化static属性
    引入成员变量初始化顺序：
    （1）默认初始化
    （2）代码块初始化
    （3）构造器初始化
    （4）对象.属性或对象.方法赋值

6.finally
在try-catch语句中不管如何都会执行
（1）例外情况，在catch中退出JVM（System.exit(0)）,finally不会执行
（2）finally不能有return，因为会覆盖try语句的返回值
（3）finally在没有return的情况下不会修改返回值
