内部类

Posted on 2020-07-22

1.成员内部类：
一般用在表示外部类的一个成员，如同人与大脑，在外部单独定义一个大脑类没必要
（1）可以访问外部类的所有成员，外部类对象也可以创建和调用内部类对象
class Outer{
private int count;
//创建内部类对象
public Inner inner = new Inner();
class Inner{
public void method(){
//内部类总有外部类引用
(Outer.this.)count = 10;
}
}
}
（2）因为是一个成员，必须先创建外部类对象，才有内部类对象
Inner inner = new Outer().new Inner();

2.静态内部类
与外部类的联系较少时，用静态内部类
（1）只能访问外部类的静态成员或方法
（2）无需外部类对象，就可以创建内部类对象
Outer.Inner inner = new Outer.Inner();

3.局部内部类
在代码块中，方法中定义内部类，使用的很少

4.匿名内部类
没有名字，用new name(){…来定义，常用于只用一次的情况
（1）name必须是某个类或接口，如Comparator比较器定制比较较为常用。
