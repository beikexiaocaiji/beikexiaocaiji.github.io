String类
1.String类的不可变性
（1）不可变性的目的
1.提高效率，字面量存在方法区常量池中，可以多个引用指向一个字面量。
2.安全。很多文件路劲，URL都是String。而且不可变天生就是线程安全
（2）不可变的含义
String是final类，不可被继承重写，确保不会有方法修改值；
实现是final char[] value的字符数组，属性被赋值就不会更改；
（3）字面量
直接放到方法区
（3）new构造器方式
构造器会在堆里创建一个对象（这一步必须要），然后常量池没有就复制一份到常量池
（5）字符串拼接（“+”或substring）
常量与常量拼接放在方法区；
有变量拼接就在堆中创建一个对象，不会放到常量池（注意final）
intern 方法会从字符串常量池中查询当前字符串是否存在，
若不存在就会将当前字符串放入常量池中，并且返回常量引用；
若存在则什么都不干。

    引入：    public static void main(String[] args) throws ExecutionException, InterruptedException {
                String s1 = "abc";
                change(s1);
                System.out.println(s1);
            }
            private static void change(String s1) {
                s1 = "abcd";
                System.out.println(s1);
            }
        值传递与字符串不变性的范例
    引入：编译期优化：final值和字面量由于编译期知道且不会再被更改，因此编译时会直接运算好且换成常量

2.StringBuffer和StringBuilder类
它们也都是final类，但它们可变 char[] value；
（1）它们的区别：
StringBuffer是线程安全，因为它的方法synchronized修饰，StringBuilder是线程不安全，所以效率更高
（2）与String的交互
也就是“+”字符串拼接，这是语法糖不是运算符重载。
本质就是String转换为StringBuffer或StringBuilder，再调用append（），最后toString。
因此for循环中，不要用“+”字符串拼接，直接用StringBuffer或StringBuilder，然后调用append（）
