JavaIO

JavaIO
1.File类
File类操作硬盘的文件及目录，注意只是对其属性，其内容由IO流决定
（1）idea中，单元测试JUnit中File对象相对路径是指模块的地址；
main()中指当前工程的地址
（2）可以作为IO（输入输出）的起点或终点
（3）使用Files，Paths（两者是工具类）和Path（替代File，两者也可以转换）扩展
Path path = Paths.get(“index.html”);

2.IO流
（1）广泛的指多种设备之间的数据传输，数据格式分字节流（适用于图片，视频等）和字符流（适用于文本）
直接从数据源与内存链接的是节点流，在其基础上装饰增加功能的是处理流
（2）IO流的类都由InputStream,OutputStream,Reader,Writer四个抽象类导出，结合具体情境使用
（3）处理流得在节点流之上创建，如下缓冲流可以提高读写速度，因为提供了缓冲区，达到一定数据量才一次性输出
File file = new File(“src/main/java/reflection/helloworld.txt”);
File fileOutput = new File(“src/main/java/reflection/hello.txt”);
char[] chars = new char[5];
int length;

    //try-with-resource语句关闭资源
    try (BufferedReader bufferedReader = new BufferedReader(new FileReader(file));
         BufferedWriter bufferedWriter = new BufferedWriter(new FileWriter(fileOutput));){
        //返回 -1 的时候表示读到 eof，即文件尾
        //返回的是实际读取的个数，注意最后读到的数组
        while ((length = bufferedReader.read(chars, 0, chars.length)) != -1){
            String s = new String(chars, 0, length);
            System.out.printf(s);
            bufferedWriter.write(s);
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
（4）转换流也是处理流的一种
    InputStreamReader：读进来字节流转换为字符流（解码）
    OutputStreamWriter:写出去字符流转换为字节流（编码）
    解码与编码的字符集不一致就会乱码
    //此处new File()可以省略
    FileInputStream fileInputStream = new FileInputStream("src/main/java/reflection/helloworld.txt");
    InputStreamReader inputStreamReader = new InputStreamReader(fileInputStream,"UTF-8");
（5）标准输入、输出流
    System.in系统标准输入，默认键盘（InputStream）
    System.out系统标准输出，默认屏幕（PrintStream，OutputStream的子类）
（6）打印流：实现将基本数据类型的数据格式转化为字符串输出
（7）数据流：用于读取或写出基本数据类型或字符串
（8）对象流
（9）RandomAccessFile：可以以直接跳到文件的任意地方来读、写文件
    因为RandomAccessFile对象包含一个记录指针，用以标示当前读写处的位置

3.网络编程
（1）socket：是IP和端口号的构成。
public void client(){
try (Socket socket = new Socket(InetAddress.getByName(“127.0.0.1”), 8899);
OutputStream outputStream = socket.getOutputStream();){
outputStream.write(“我是客户端”.getBytes());
} catch (IOException e) {
e.printStackTrace();
}
}
public void server(){
try ( ServerSocket serverSocket = new ServerSocket(8899);
Socket accept = serverSocket.accept();
InputStream inputStream = accept.getInputStream();) {
//之所以使用ByteArrayOutputStream,做一个缓冲区，避免读取字节分割造成乱码
ByteArrayOutputStream arrayOutputStream = new ByteArrayOutputStream();
byte[] bytes = new byte[5];
int length;
while ((length = inputStream.read(bytes, 0, bytes.length)) != -1){
arrayOutputStream.write(bytes, 0, bytes.length);
}
System.out.println(arrayOutputStream.toString());
} catch (IOException e) {
e.printStackTrace();
}
}
注意先开启服务端，才能接受客户端

4.NIO
（1）与IO的区别：
NIO面向缓冲区，IO面向流。流是单向的，NIO会建立Channel（IO设备的连接，相当于轨道）
缓冲区用于存储数据往返（相当于火车），是双向的
（2）缓冲区
Buffer底层是数组，根据数据类型不同使用不同子类。
创建缓冲区：ByteBuffer buffer = ByteBuffer.allocate(48);，
容量：capacity,Buffer的大小，创建就不能更改
限制：limit，之后索引的数据不能操作，读写模式更改会改动
位置：下一个读写数据的索引
获取：get()、放入：put()、清空clear()、读模式flip()
标记：mark,记录索引，重置：reset,可以使位置到标记处
（3）直接缓冲区与非直接缓冲区
非直接缓冲区：程序与磁盘读写要经历用户地址空间（JVM堆内存）和内核地址空间（OS内存）
allocate()和传统IO分配都是非直接缓冲区。
直接缓冲区：使用物理内存映射文件（堆外内存），减少一次拷贝，速度提高，allocateDirect()分配
局限是创建销毁开销大，而且回收时间不确定
（4）通道
与流的区别：通道可以双向，流只能单向；通道非阻塞，流阻塞；通道传输必须通过缓冲区，流可以选择性
分散读取：通道数据分散给多个缓冲区；聚集写入：多个缓冲区按顺序写入同一个通道
（5）阻塞和非阻塞
指的是程序在等待调用结果（消息，返回值）时的状态
阻塞什么都不干，非阻塞可以干别的
（6）选择器
选择器是通道的多路复用器，是非阻塞NIO的核心（注意FileChannel不能注册，因为选择器用于网络传输）

5.BIO、NIO、AIO
（1）BIO同步阻塞：服务端一个线程对应客户端一个链接，这样并发量大的时候服务器会宕机。
而且当链接不做任何事情时线程会等待进入阻塞状态，消耗资源。
（2）线程池改进。引入线程池，分配一定的线程数，这样就避免服务器宕机，而且减少线程创建和销毁
的开销，但是本质还是同步非阻塞
（3）NIO同步非阻塞：将多个客户端通道注册到选择器中，选择器监视链接的状态，轮询到有IO请求时，
才启动线程处理，因此单个线程能够处理多个通道。
（4）AIO异步非阻塞：需要得到OS支持，OS处理完客户端请求后再发消息通知服务端
